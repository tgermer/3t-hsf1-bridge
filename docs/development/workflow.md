# Development Workflow

For every feature or refactoring:

1. Create an Issue
2. Add labels
3. Add to Project
4. Update architecture documentation (if required)
5. Update learning documentation (if we learned something new)
6. Implement the change
7. Build the project
8. Test the feature
9. Update CHANGELOG (if applicable)
10. Commit using `Closes #...`
11. Push to GitHub
12. Close the Issue automatically
